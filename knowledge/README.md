# Project knowledge

This folder holds **living notes** about the game: design intent, architecture, conventions, and what changed recently. Update these files as the project grows so context stays easy to hand off (to collaborators or to future you).

## How to use

- **`PROJECT.md`** — Current snapshot: engine version, structure, main scenes/scripts, and open questions. Refresh when milestones land.
- Add more files as needed (e.g. `design/`, `systems/`, `decisions.md` for ADRs). Keep names obvious.

## Suggested rhythm

- After meaningful changes: skim `PROJECT.md` and adjust the snapshot section.
- When choosing something non-obvious: one short note in a decisions doc or inline in `PROJECT.md`.
